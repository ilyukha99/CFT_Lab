Для того, чтобы запустить программу, имея файлы с кодом, нужно проделать ниже лежащую инструкцию.
0) Убедиться, что java установлена. Если нет, то установить jdk.
java -version

1) Указать все файлы для компилятору и применить javac
javac Main.java ... -опции 

2) Создать jar-архив при помощи jar cfe, указав по порядку имя jar-файла, имя главного класса, затем все файлы *.class.
jar cfe TestName.jar MainClass MainClass.class ...

3) Запуск программы при помощи java -jar
java -jar TestName.jar -ключи выходной_файл входные_файлы

(или просто-напросто собрать артефакт-jarник в IntelliJ IDEA)

Особенности реализации.
Особенностью является пятый возможный ключ, -bf. Данный ключ применяется на усмотрение пользователя в том случае, когда он имеет 
дело с файлами внушительного размера. Программа же для себя определяет файл большИм, если его объём превосходит 1/10 от 
доступной оперативной памяти с некоторой поправкой на внезапные затраты памяти процессором. В этом случае файлы небольшого объёма 
считаются меньше 1/10, части памяти поэтому не рекомендуется прописывать более 10 in-файлов внушительных относительно числа оперативной 
памяти размеров, которые потенциально могут быть определены файлами небольшого объёма.
Также стоит отметить что большие файлы, не являющиеся в свою очередь отсортированными, будут отброшены программой(c -bf) при попытке их
слить сортировкой с другими файлами. Было принято такое решение, так как выборочная сортировка файлов способна терять данные, что может
быть недопустимо при получении результата.

Если пользователь считает, что файлы невелики по размеру относительно числа доступной оперативной памяти в данный момент (сотые/тысячные
доли и меньше), то предполагается не использовать ключ -bf. Этот вариант должен иметь преимущество по скорости перед вторым в такой 
ситуации.
